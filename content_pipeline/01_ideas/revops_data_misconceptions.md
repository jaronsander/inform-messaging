# Idea: RevOps Data Misconceptions

**Date Captured:** 2026-02-17
**Theme:** Thought Leadership — Data Quality / Source of Truth
**Type:** B (Expert Opinion)
**Status:** In Draft

---

## Raw Notes

Biggest misconceptions about RevOps data:

1. **Marketing as source of truth** — Web/marketing data is never 100% accurate due to browsers, GDPR, extensions. Treat it as ratios and trends, not absolute numbers. True source of truth happens when someone identifies themselves (form submission, enters CRM/database).

2. **CRM data is complete** — Different integrations and tools fill different fields. Contacts from different motions have different data. Hard to normalize and compare. Example: employee count ranges vary by enrichment tool (51-200 vs 51-100 vs 51-500).

3. **Customer success data is honest** — Hardest to get honest data from customers. People take the shortest path, want to please you, want to save time. Most data collected is biased. Getting unbiased honest customer data takes real effort on the customer side.

4. **Finance is the ultimate source of truth** — Finance is closest to truth but constrained by time scales. Customer value and transactions are bound by months, quarters, years. Those distinct cutoffs can distort the picture.

---

## Core Insight

Each data layer in the revenue stack is unreliable in a *different* way. Most teams don't know the failure mode of each layer — so they misplace confidence. The only true source of truth is a human identifying themselves.
